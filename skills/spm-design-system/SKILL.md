# SPM Design System — 视觉设计规范

> 不教设计理论。只告诉 Agent"你这个项目应该长什么样"，避免 AI 画出紫粉渐变+Inter字体的千篇一律页面。

## 自动检测

SPM Phase 2 写 WBS 计划时，根据 spec 中的项目类型自动引用本技能，将设计 token 注入到每个 Context Brief 的"关键约束"中。

---

## 设计工作流

```
1. 识别项目类型 → 匹配下方分类
2. 提取该类型的配色/字体/布局/反模式 → 形成设计 token 卡
3. 在每个 WBS 任务的 Context Brief 中注明关键 token
4. 前端子代理读 Context Brief → 按 token 实现
```

## 8 种项目类型

### 1. SaaS / 工具类

| Token | 推荐 |
|-------|------|
| 主色 | 蓝色系 (`#2563EB`-`#3B82F6`) 或中性深色 |
| 背景 | `#FAFAFA` 浅灰白, `#1A1A2E` (暗色) |
| 字体 | Inter / SF Pro (系统原生感) |
| 间距 | 紧凑 16px/24px/32px，信息密度高 |
| 圆角 | 6-8px，卡片统一 |
| 阴影 | 1-2 层浅阴影，不超 `0 2px 8px rgba(0,0,0,0.08)` |
| 布局 | 侧边栏 + 主内容区 |
| 动效 | 150ms ease，hover 微微上浮 2px |
| ❌ 反模式 | 大色块、花哨渐变、emoji 做图标、圆角 > 12px |

### 2. 开发者工具 / CLI

| Token | 推荐 |
|-------|------|
| 主色 | 终端绿 (`#00FF41`) / 琥珀 (`#FFB000`) / 纯黑白 |
| 背景 | `#0D1117` 深色，`#FFFFFF` 浅色 |
| 字体 | JetBrains Mono / Fira Code (代码), Inter (正文) |

| 圆角 | 4-6px |
| 阴影 | 极少或无阴影 |
| 布局 | 单栏 / 文档式 |
| 动效 | 无或极简 |
| ❌ 反模式 | 大图 hero、圆角按钮、渐变背景、emoji |

| 圆角 | 0-4px |

### 6. Dashboard / 数据

| Token | 推荐 |
|-------|------|
| 主色 | 深底色 (`#0F172A` / `#1E293B`)，数据色 (绿涨红跌) |
| 背景 | 深色 (#0F172A) 或浅灰白 |
| 字体 | Inter / system-ui |
| 间距 | 紧凑，充分利用屏幕 |
| 圆角 | 6-8px |
| 阴影 | 无，用边框替代 |
| 布局 | 网格 Widget，响应式 |
| 动效 | 图表动画 300ms ease-out |
| ❌ 反模式 | 大面积留白、大圆角卡片、hero 图、emoji |

### 7. 健康 / 医疗

| Token | 推荐 |
|-------|------|
| 主色 | 蓝绿 (`#0891B2` / `#059669`) / 柔和暖色 |
| 背景 | 暖白 `#FFFBF5`，柔和浅蓝 |
| 字体 | 圆体友好感 (Nunito) + Inter |
| 间距 | 宽松，不拥挤 |
| 圆角 | 12-16px，圆润友好 |
| 阴影 | Soft UI，极浅 |
| 布局 | 卡片式，单列 |
| 动效 | 200ms ease，柔和 |
| ❌ 反模式 | 纯白惨淡背景、刺眼蓝色、尖锐圆角、暗色模式 |

### 8. 创意 / 作品集

| Token | 推荐 |
|-------|------|
| 主色 | 大胆单色或黑白 |
| 背景 | 大胆对比，黑白/全屏大图 |
| 字体 | 有个性的展示字体 + 无衬线 |
| 间距 | 大留白，呼吸感 |
| 圆角 | 0 或 >16px，两端 |
| 阴影 | 大胆或无 |
| 布局 | 非标准，全屏滚动 |
| 动效 | 视差、滚动触发、Lenis 平滑滚动 |
| ❌ 反模式 | 通用模板布局、保守配色、信息过多 |

## 通用反模式（所有类型）

这是 AI 写前端时**最常见的视觉问题**，不管什么项目类型都要避免：

| # | 反模式 | 原因 |
|---|--------|------|
| 1 | **紫粉渐变背景** (`linear-gradient(135deg, #667eea, #764ba2)`) | AI 最爱的默认审美，千篇一律 |
| 2 | **Inter + 紫蓝配色** | 又一个 AI 默认组合 |
| 3 | **圆角彩色左边框卡片** | "看起来像 AI 写的"的标志 |
| 4 | **emoji 当图标** (`🚀 ⚡ ✨ 🔥`) | 用 SVG 图标替代 |
| 5 | **假数据填充** ("10,000+ 用户" / "99.9% 可用性") | 用占位符 `[data]`，不要造假 |
| 6 | **所有东西都是卡片** | 用间距和排版分层，不要套卡片 |
| 7 | **假 logo 墙** | 用 `[logo]` 占位，不要编客户名字 |

## 与 SPM 集成

SPM Phase 2 写 Context Brief 时：

```
Context Brief: 实现 Dashboard 首页。
前置: Task 1 已完成项目脚手架 (Vite + React + Tailwind)。
涉及: src/pages/Dashboard.tsx, src/components/WidgetCard.tsx。
设计 token: SaaS/工具类 — 蓝色主色, Inter, 紧凑间距, 6px圆角, 侧边栏布局。
❌ 禁止: 紫粉渐变, emoji图标, 假数据, 大圆角卡片。
```
